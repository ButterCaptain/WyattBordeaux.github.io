Overview

This project is a complete, low-latency attitude estimation and telemetry pipeline modeled directly after real aerospace ground stations. It utilizes two ESP32 microcontrollers, an MPU6050 IMU, and a custom Node.js/WebSocket bridge to render a live 3D visualization in the browser. By bypassing standard Wi-Fi and IP networks entirely, the system achieves a highly robust, zero-latency data link.

Problem & Motivation

When attempting to stream sensor telemetry over standard HTTP/Wi-Fi, the update rate capped at an unusable 2–4 Hz due to TCP handshake and JSON serialization overhead. My motivation was to solve this latency bottleneck by engineering a true connectionless, peer-to-peer radio link. I wanted to move beyond basic microcontroller projects and build an end-to-end architecture that mirrors how real flight vehicles communicate with ground control.

Technical Details

The system relies on a custom C++ firmware architecture that eliminates serialization overhead in favor of raw binary transmission. Data flows seamlessly from an I2C sensor to a connectionless radio link, over USB serial, and finally to a browser-based Three.js dashboard.

  • Sensor Fusion: Implemented the Madgwick AHRS filter to fuse accelerometer and gyroscope data into a gimbal-lock-free quaternion.
  • Binary Telemetry Link: Utilized ESP-NOW and #pragma pack(1) memory alignment to transmit a perfectly packed 16-byte struct, bypassing the Wi-Fi stack entirely.
  • Data Parsing: Engineered a Node.js hardware bridge that hunts for a custom 0xAA 0xBB sync beacon to extract clean packets from a continuous, raw USB serial stream.

What I Learned

This project was a deep dive into the limitations of standard web protocols for real-time hardware applications. I gained significant hands-on experience with embedded sensor fusion algorithms, memory alignment, and writing custom parsers for raw binary data streams. Ultimately, it taught me how to think systematically about latency, network overhead, and data integrity when designing complex embedded systems.

Next Steps

Moving forward, I plan to integrate a magnetometer to transition the system to a full 9-DOF AHRS, providing an absolute yaw reference. I also intend to implement an adaptive gain parameter to automatically adjust filter weights during high-G linear acceleration maneuvers.
