Microcontroller-Based RFID Access Control System

A contactless, Arduino-driven RFID security system designed to replace traditional cylinder locks.

July 2025 Completed
GitHub

Overview

Funded by Wrebotics during my internship, this project is a DIY contactless security system kit designed to teach young engineers the fundamentals of embedded systems. By building an Arduino-driven RFID lock, students learn hardware integration, C++ programming, and electromechanical control in a hands-on, accessible way.

Problem & Motivation

I noticed that traditional STEM kits often lack practical, real-world applications that truly keep students engaged and excited to learn. My motivation was to bridge this gap by designing a kit where kids build a tangible, useful product—a smart lock—while safely mastering complex hardware concepts. Ultimately, I wanted to empower the next generation of creators to explore engineering with confidence and curiosity.

Technical Details

The system architecture centers around an Arduino Uno, programmed in C++ to manage both hardware interfacing and the access control logic. To optimize pin usage and introduce students to different communication standards, I utilized the SPI protocol for the MFRC522 RFID reader alongside the I2C bus for the 16x2 LCD display.

  • Multi-Protocol Integration: Leveraged both SPI and I2C buses simultaneously to maximize the microcontroller's efficiency without hardware conflicts.
  • Dynamic Firmware Logic: Developed C++ routines to handle UID authentication, including a startup sequence that sets the first scanned tag as a configurable "Master Key."
  • Electromechanical Actuation: Calibrated specific PWM signals to drive a servo motor, simulating a precise 90-degree physical latching mechanism.

What I Learned

This project deepened my technical proficiency in embedded C++ and low-level hardware communication, but more importantly, it taught me how to translate complex engineering systems into accessible, student-friendly milestones. Designing for education forced me to prioritize clean, modular code and robust hardware integration so that younger users wouldn't be overwhelmed.

Next Steps

I plan to improve the kit by introducing EEPROM storage to retain key data after power loss and implementing interrupt-based sleep modes to extend battery life.
